Generated by /plan-design-review on 2026-04-26.

## Typography

```css
:root {
  /* Display serif — chapter titles, work titles, drop cap */
  --font-display: "Tiempos Headline", "Source Serif 4 Display", Georgia, serif;
  /* Body serif — chapter content, marginalia */
  --font-body: "Crimson Pro", "Source Serif 4", Georgia, serif;
  /* Sans — chrome, nav, byline, UI */
  --font-sans: "Inter", "IBM Plex Sans", -apple-system, sans-serif;
  /* Mono — citation metadata, code-ish spans */
  --font-mono: "IBM Plex Mono", "JetBrains Mono", ui-monospace, monospace;
  /* Nastaliq — Urdu original lines (verse layout) */
  --font-nastaliq: "Noto Nastaliq Urdu", serif;
  /* Devanagari — for Sanskrit transliteration when needed */
  --font-devanagari: "Noto Serif Devanagari", serif;
}
```

4 / 8 / 12 / 16 / 24 / 32 / 48 / 64 / 96 / 128 px

- Section vertical rhythm: 64px desktop, 48px mobile
- Body line spacing: 1.7 × font-size

## Reader preferences panel

Top-right gear icon on chapter pages opens a small panel:

| Setting | Options | Default |
|---------|---------|---------|
| Size | S (16px body) / M (19px) / L (22px) | M |
| Density | Cozy (line-height 1.5) / Comfortable (1.7) / Spacious (1.9) | Comfortable |
| Theme | Warm light / Dark / Sepia | Warm light |

Stored in localStorage. Applied via `data-size`, `data-density`, `data-theme`
attributes on `<html>`.

## Reduced motion

`@media (prefers-reduced-motion: reduce)`: disable view transitions,
reading-progress pulse, drawer slide animations. Use instant open/close
and crossfade-only transitions instead.

## Drop cap implementation

```css
.chapter-content::first-letter {
  font-family: var(--font-display);
  font-weight: 800;
  font-size: 5.4em;
  line-height: 0.85;
  float: left;
  margin: 0.05em 0.08em 0 0;
  color: var(--accent);
}
```

Drop cap renders only when `layout: prose` AND `::first-letter` selector
applies (some browsers handle Unicode poorly — fallback: no drop cap, use
`<span class="drop-cap">A</span>` for guaranteed control).

For `layout: verse` and `layout: manuscript`: NO drop cap.

## What's not in this document

The visual identity is intentionally minimal here. Refinement comes from:
- `/design-consultation` for typography refinement, additional palette work,
  motion design, illustration style
- Approved mockups (referenced above) — visual reference for layout details
- Implementation iterations — this doc captures intent; real CSS will refine

When in doubt, lean editorial. NYRB doesn't have a single piece of decoration
without intent. Neither should Falsafa.
